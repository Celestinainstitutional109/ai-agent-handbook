# Contributing

Thanks for your interest in making this guide better. Every contribution helps the community build better agents.

## How to Contribute

### Quick fixes
Typos, broken links, outdated star counts — just open a PR. No issue needed.

### New framework analysis
If you've dug into a framework that isn't covered (or one that's changed significantly):

1. Fork the repo
2. Add your analysis following the existing format in `AI_AGENT_FRAMEWORKS_2026_ANALYSIS.md`
3. Update the framework table in `README.md`
4. Open a PR with a brief description of what you analyzed

### New patterns or insights
Discovered a production pattern that isn't documented here? Found a better way to handle context rot, tool sprawl, or multi-agent coordination?

1. Open an issue describing the pattern
2. Include code examples or references to real implementations
3. We'll work with you to integrate it into the guide

### Corrections
If something is wrong — a mischaracterized architecture, an outdated API, a pattern that doesn't work as described — please open an issue. Getting the details right matters more than anything.

## Guidelines

- **Be specific.** "This section is wrong" isn't actionable. "Section 14 says CrewAI uses X but since v0.80 it uses Y" is.
- **Show your work.** Link to source code, docs, or benchmarks when possible.
- **Keep the tone.** Direct, technical, no fluff. Every line should deliver value.
- **Test code examples.** If you're adding code, make sure it runs.

## What We're Looking For

- Framework updates (new versions, architecture changes)
- Production war stories (what broke, what fixed it)
- New patterns emerging in the ecosystem
- Benchmark results and comparisons
- Translations (open an issue to coordinate)

## Code of Conduct

Be respectful, be constructive, be helpful. We're all here to build better agents.
